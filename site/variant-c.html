What we teach

Engineering discipline. Not prompt tricks.

  1. PrototypeState the question. Build the smallest thing that answers it. If you can't write down what you're answering, you're decorating, not prototyping.
  2. DesignData shape, user shape, failure shape. The data model is the contract; the interface is the hypothesis.
  3. BuildAI-assisted, but you own every line. Claude writes; you read, edit, decide. Velocity is a side-effect of judgment.
  4. DeployTo one user, not to staging. "Live" means a person outside your laptop is using it. Anything less is rehearsal.
  5. SupportThe product isn't done until someone keeps using it. Retention is the only metric that doesn't lie.
