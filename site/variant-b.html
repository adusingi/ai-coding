##What we teach

Engineering discipline with AI. Not prompt tricks.

  • Prototype — state the question; build the smallest thing that answers it. If you can't write down what you're answering, you're not prototyping — you're decorating.
  • Design — data shape, user shape, failure shape. The interface is a hypothesis. The data model is the contract.
  • Build — AI-assisted, but you own every line. Claude writes; you read, edit, decide. Velocity is a side-effect of judgment.
  • Deploy — to one user, not to staging. "Live" means someone outside your laptop is using it. Anything less is rehearsal.
  • Support — the product isn't done until someone keeps using it. Retention is the only metric that doesn't lie. Talk to the user, fix the thing, repeat.
